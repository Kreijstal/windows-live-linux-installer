      Entering Custom Code









The Custom Code screen will appear when you select an OS Type of isoboot or custom code or if you select the Load The Kernel By Custom Code option on the Editing OS details screen. It allows you to enter your own Grub commands instead of having the program generate them.


Custom code can be useful to set up operating systems not supported by Grub2Win or if you simply want to enter your own commands. You can also enter Grub diagnostic or utility commands such as ls, sleep and reboot.



Menu Slot

Changes the order of the selections displayed in the Grub boot menu



Title

The text that will be displayed by Grub for this OS in the Grub boot menu selection. It must be 1 to 80 characters.



Click The Icon

Click on the icon image. This takes you to the Change Icon screen where you can set the icon that is displayed at boot time.



Graphics Payload

The graphics mode Grub will establish for the target OS when it is selected for boot.



Pause Seconds

Grub will pause for the specified number of seconds before booting the target OS. This will allow you time to read any messages from Grub, such as incorrect partition address or label name. Set to zero seconds if you don't want Grub to pause. Also sets the $reviewpause variable at boot time



Edit Custom Code

This will start the Windows Notepad program, which will allow you to edit your code. When you save the file and quit Notepad, your code will be checked for basic syntax errors. It will then be transferred to the Grub2Win configuration and will be shown in the window.




Load Sample Code

Loads a few lines of Grub commands tailored for the OS you are defining. You can then edit the code to suit your needs.
